Wörterbuch

Hier findest du eine Sammlung von Fachbegriffen rund um das Thema Modding.

Dieses Wörterbuch wurde von mir und meinem Kumpel erstellt, um Modding-Begeisterten zu helfen, den Überblick zu behalten.

Viel Spaß beim Entdecken der Begriffe :D

Suchbegriff eingeben:

Homebrew

Homebrew bezeichnet Software, die von unabhängigen Entwicklern erstellt wurde, um auf einer Konsole ausgeführt zu werden. Sie ist nicht von den offiziellen Herstellern wie Nintendo, Sony oder Microsoft autorisiert. Homebrew-Software kann viele verschiedene Zwecke haben, darunter Emulatoren, Medienplayer, Backup-Manager und modifizierte Spiele.

Beispiel: Ein Homebrew-Emulator ermöglicht es, ältere Spiele auf einer modernen Konsole zu spielen.

Jailbreak

Ein Jailbreak ist der Prozess, bei dem Sicherheitsmechanismen einer Konsole umgangen werden, um nicht offiziell signierte Software auszuführen. Dies kann über Software-Exploits oder durch den Einsatz von Hardware-Modifikationen (z. B. Modchips) geschehen.

Risiken: Ein Jailbreak kann Garantieverlust bedeuten, Updates können das Gerät sperren, und unsachgemäße Modifikationen können zu einem Brick führen.

Nutzen: Ermöglicht die Installation von Homebrew, Emulatoren, benutzerdefinierten Themes und manchmal auch das Abspielen von Backups.

Custom Firmware (CFW)

Custom Firmware (CFW) ist eine modifizierte Version der Original-Firmware einer Konsole, die zusätzliche Funktionen und Freiheiten bietet. Während die Standard-Firmware vom Hersteller meist eingeschränkt ist, erlaubt eine Custom Firmware die Nutzung von Homebrew-Software, Emulatoren und oft auch Sicherheitskopien von Spielen.

Beispiele für bekannte CFWs:

  • Atmosphère (Nintendo Switch)
  • Luma3DS (Nintendo 3DS)
  • GoldHen (PlayStation 3)

Unterschied zu einem Jailbreak: Ein Jailbreak ist oft der erste Schritt, um eine Custom Firmware zu installieren.

Modchip

Ein Modchip ist eine physische Hardware-Komponente, die in eine Konsole eingebaut wird, um sie zu modifizieren. Modchips dienen oft dazu, Systembeschränkungen zu umgehen und Homebrew oder nicht autorisierte Software auszuführen.

Arten von Modchips:

  • Softmodding: Softwarebasierte Exploits, die keine Hardware-Änderung erfordern.
  • Hardmodding: Das physische Einlöten eines Modchips in die Hardware.

Beispiele für Konsolen mit Modchips:

  • Nintendo Switch (HWFLY, SX Core)
  • PlayStation 2 (Matrix Infinity)
  • Xbox 360 (RGH - Reset Glitch Hack)

Risiken: Beim Einlöten eines Modchips kann die Konsole beschädigt werden, und einige Hersteller sperren modifizierte Konsolen online.

Brick

Ein "Brick" (engl. für "Ziegelstein") beschreibt eine Konsole, die durch eine fehlerhafte Modifikation oder ein fehlgeschlagenes Firmware-Update unbrauchbar geworden ist.

Arten von Bricks:

  • Soft Brick: Die Konsole startet nicht richtig, kann aber oft durch Software-Werkzeuge wiederhergestellt werden.
  • Hard Brick: Die Konsole reagiert gar nicht mehr und benötigt meistens Hardware-Reparaturen.

Ursachen für einen Brick:

  • Unsachgemäße Installation von Custom Firmware
  • Abbruch eines Firmware-Updates
  • Verwendung inkompatibler Homebrew-Software

Wie vermeidet man einen Brick?

  • Eine vollständige NAND-Backup erstellen, bevor man Modifikationen durchführt.
  • Nur vertrauenswürdige Software von verlässlichen Quellen verwenden.
  • Niemals System-Updates durchführen, wenn eine Custom Firmware aktiv ist (ohne vorherige Prüfung der Kompatibilität).

Exploit

Ein Exploit ist eine Schwachstelle in der Software oder Hardware einer Konsole, die genutzt werden kann, um nicht autorisierten Code auszuführen. Exploits sind oft der erste Schritt zum Jailbreak oder zur Installation von Custom Firmware.

Beispiele für bekannte Exploits:

  • FreakyHax (Nintendo 3DS - nutzt einen Fehler in Freakyforms Deluxe)
  • Fusée Gelée (Nintendo Switch - nutzt eine Sicherheitslücke im Boot-ROM)
  • PS4 Kernel Exploits (PlayStation 4 - verschiedene Exploits je nach Firmware-Version)

Risiken: Einige Exploits können durch Updates des Herstellers geschlossen werden.

Payload

Ein Payload ist eine kleine Software-Datei, die über einen Exploit oder einen speziellen Bootloader auf einer Konsole ausgeführt wird. Payloads enthalten oft Anweisungen, um eine Custom Firmware zu starten oder Systembeschränkungen zu umgehen.

Beispiele für Payloads:

  • Hekate (Nintendo Switch - Multi-Tool für NAND-Backups, Bootloader etc.)
  • Fusee Primary (Nintendo Switch - Startet Atmosphère CFW)

NAND

Der NAND-Speicher ist der interne Speicher einer Konsole, in dem wichtige Systemdaten und die Firmware gespeichert sind. Modder erstellen oft ein vollständiges NAND-Backup, bevor sie eine Custom Firmware installieren, um eine Wiederherstellung im Falle eines Bricks zu ermöglichen.

Wichtige Begriffe im Zusammenhang mit NAND:

  • NAND-Dump: Eine exakte Kopie des internen Speichers der Konsole.
  • EmuNAND: Eine emulierte Version des NANDs, die von einer SD-Karte ausgeführt wird, um das echte System zu schützen.

Reset Glitch Hack (RGH)

RGH ist eine Methode, um die Xbox 360 zu modifizieren, indem gezielt Glitches im Boot-Prozess genutzt werden, um nicht signierten Code auszuführen.

Vorteile:

  • Ermöglicht das Starten von Homebrew und Custom Firmware.
  • Kann auf fast allen Xbox 360-Modellen durchgeführt werden.

Nachteile:

  • Aufwendige Hardware-Modifikation erforderlich.
  • Die Bootzeit kann je nach Konsole variieren.

Softmod

Ein Softmod ist eine Modifikation, die ausschließlich über Software durchgeführt wird, ohne dass physische Hardware geändert werden muss. Softmods nutzen oft Exploits oder Sicherheitslücken im Betriebssystem der Konsole.

Beispiele für bekannte Softmods:

  • LetterBomb (Nintendo Wii - nutzt eine Sicherheitslücke in der Nachrichten-App)
  • Haxchi (Nintendo Wii U - nutzt eine Virtual Console-Anwendung für den Exploit)
  • Frogminer (Nintendo 3DS - nutzt einen DS-Download-Spiel-Exploit)

Hardmod

Ein Hardmod ist eine Modifikation, die physische Änderungen an der Hardware einer Konsole erfordert, z. B. das Einlöten eines Modchips oder das direkte Auslesen des NAND-Speichers.

Beispiele für Hardmods:

  • Modchips für die PS2 - Ermöglichen das Abspielen von importierten oder Sicherheitskopien von Spielen.
  • HWFLY für die Switch - Ermöglicht die automatische Ausführung von Payloads beim Einschalten.

Emulator

Ein Emulator ist eine Software, die eine andere Hardware-Umgebung simuliert, sodass Konsolenspiele auf einem PC oder einer anderen Konsole ausgeführt werden können.

Beispiele für Emulatoren:

  • Dolphin - Ein GameCube- und Wii-Emulator für PC.
  • Citra - Ein Nintendo 3DS-Emulator.
  • PCSX2 - Ein PlayStation 2-Emulator.

BIOS

Das BIOS (Basic Input/Output System) ist eine Firmware, die als grundlegende Steuerung für eine Konsole dient und beim Starten der Hardware geladen wird.

Wichtige Informationen zum BIOS:

  • Viele Emulatoren benötigen die originale BIOS-Datei, um korrekt zu funktionieren.
  • Das Flashen eines falschen BIOS kann zu einem Brick führen.

SysNAND

SysNAND ist der echte, interne NAND-Speicher der Konsole, auf dem das offizielle Betriebssystem läuft.

Unterschied zwischen SysNAND und EmuNAND:

  • SysNAND: Das echte System der Konsole.
  • EmuNAND: Eine virtuelle Kopie von SysNAND, die auf einer SD-Karte gespeichert wird.

EmuNAND

EmuNAND ist eine emulierte Version des NAND-Speichers einer Konsole, die von einer SD-Karte oder einem externen Speicher ausgeführt wird.

Vorteile von EmuNAND:

  • Ermöglicht die Nutzung von Homebrew ohne Risiko für das echte System.
  • Updates können auf EmuNAND getestet werden, bevor sie auf SysNAND angewendet werden.
